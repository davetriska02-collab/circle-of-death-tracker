Storage Mode Switch

      

      

        

          

            The storageMode key in config.json controls how sessions are
            submitted. Two modes are supported:
          

          

            "worker" — Sessions are POSTed to the Cloudflare Worker URL specified
            in workerUrl. The Worker holds the GitHub PAT as a secret and creates the
            Issue on behalf of the user. This is the recommended mode for production because no
            secrets are stored on the client device.
          

          

            "pat" — The app reads a PAT from localStorage (set via
            the Fallback PAT Mode section above) and calls the GitHub API directly from the browser.
            Use this only for testing or when a Worker is not available.
          

          

            Changing storageMode requires editing config.json and
            committing it to the repository. See the Worker setup guide for deployment instructions:
            ../worker/README.md.
